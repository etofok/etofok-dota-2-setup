Actually useful fun fact!

You can make the minimap even bigger.
It's possible to enable "Extra Extra Large Minimap" with this console command:

  • dota_hud_extra_large_minimap 2

The minimap is about 420x420 size on my 1920x1080 screen.
And after playtesting this extensively I honestly can't go back.

It's so much easier on the eyes and the smooth high quality feels very nice.
So I'm currently using it.

There's a big problem with it:
Because it's not a supported feature, this gigantic minimap obscures the event log on the left-hand side of your screen.

The event log is behind the minimap.

But there's a workaround I got from the user C6500:

Bind a key to hotswitch between the different sizes, like so:

  • bind "F4" "toggle dota_hud_extra_large_minimap 1 2"

This way "F4" toggles between "dota_hud_extra_large_minimap 1" and "dota_hud_extra_large_minimap 2".

Minimap Misclick Protection Time

This one is interesting, because it depends on your skill level.

Newbie players often misclick on the minimap in panic.

This notably happens to Radiant players intending to walk back to base, but they overshoot their mouse movement and accidentally click on the minimap.
This often makes them walk in whatever direction and feed.

That's exactly what this option is for:
It ignores all your minimap inputs for a split second to stop this from happening.

You get more value out of this setting if your minimap is big.
You get more value out of this setting if you're very new to PC peripherals such as the mouse.

Unfortunately, for more experienced players this setting makes it worse, because the game literally doesn't register the clicks.

Which is the opposite of what you want when you click on things.

So that's bad.

Set it to 0 unless absolutely necessary.
